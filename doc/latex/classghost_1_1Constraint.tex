\subsection{Detailed Description}
This class encodes constraints of your C\+S\+P/\+C\+OP. 

You cannot directly use this class \hyperlink{classghost_1_1Constraint}{Constraint} to encode your C\+S\+P/\+C\+OP constraints, since this is an abstract class. To model a problem with G\+H\+O\+ST, you have to make your own constraints by inheriting from this class. The same problem can be composed of several subclasses of \hyperlink{classghost_1_1Constraint}{Constraint}.

The only pure virtual \hyperlink{classghost_1_1Constraint}{Constraint} function is required\+\_\+cost.

Pure virtual function to compute the current cost of the constraint. 

This function is fundamental\+: it evalutes how much the current values of variables violate this contraint. Let\textquotesingle{}s consider the following example \+: consider the contraint (x = y).~\newline
If x = 42 and y = 42, then these values satify the contraint. The cost is then 0.~\newline
If x = 42 and y = 40, the constraint is not satified, but intuitively, we are closer to have a solution than with x = 42 and y = 10,000. Thus the cost when y = 40 must be strictly lower than the cost when y = 10,000.~\newline
Thus, a good required\+\_\+cost candidate for the contraint (x = y) could be the function $\vert$x-\/y$\vert$.

This function M\+U\+ST returns a value greater than or equals to 0.

\begin{DoxyWarning}{Warning}
Do not implement side effect in this function. It is called by the solver to compute the constraint cost but also for some inner mechanisms (such as cost simulations).
\end{DoxyWarning}
\begin{DoxyReturn}{Returns}
A positive double corresponding to the cost of the constraint with current variable values. Outputing 0 means current values are satisfying this constraint. 
\end{DoxyReturn}
